# Algoshift: Technical Definition

    ## Formal Definition
    Labor scheduling where algorithms dynamically adjust worker hours based on real-time demand predictions, creating unpredictability and externalizing scheduling costs onto workers.

    ## Conceptual Background
    Algoshift is used here as an operational concept rather than a rhetorical label. It addresses algorithm-driven schedule volatility in labor allocation. The mechanism focus is high algorithmic control amplifying shift instability and income uncertainty, which can be represented with explicit channels: demand_uncertainty, algorithmic_control, shift_variability, worker_autonomy, schedule_notice_hours, income_volatility, turnover_risk. This repository couples concept definition with executable implementation so assumptions remain inspectable. In practice, the framework supports comparative statics, scenario simulation, and policy diagnostics. Typical applications include gig and retail scheduling analysis, labor standards impact assessment, and workforce retention modeling. Because the index logic is transparent, researchers can vary weights and test robustness across model specifications.
